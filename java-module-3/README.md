# java-module-3

Модуль 3 — **ООП**. Сборник из 45 учебных задач: каждая задача — это отдельный класс-модель данных (`TaskNN`) с полями, конструктором, геттерами/сеттерами, валидацией и переопределёнными `toString`/`equals`/`hashCode`, плюс набор тестов JUnit 5 (`TaskNNTest`).

Описание каждой задачи дано в Javadoc внутри класса **на украинском и русском языках**.

## Требования

- JDK 21
- Maven 3.9+

## Сборка

```bash
mvn -q compile
```

## Запуск всех тестов

```bash
mvn test
```

Изначально все тесты падают красным, потому что тело каждого метода (включая конструктор) содержит:

```java
// TODO: реализуй
throw new UnsupportedOperationException("TODO: реализуй");
```

## Запуск тестов одной задачи

```bash
mvn test -Dtest=Task07Test
```

Можно несколько сразу:

```bash
mvn test -Dtest=Task01Test,Task02Test
```

## Как работать с TODO

1. Открой проект в IntelliJ IDEA как Maven-проект (`File → Open`, выбрать `pom.xml`).
2. Запусти `mvn test` — все тесты красные.
3. Возьми любой красный тест, например `Task07Test`.
4. Открой соответствующий `TaskNN.java` в `src/main/java/com/course/tasks/`.
5. Прочитай задание в Javadoc (укр/рос) и реализуй: убери `throw new UnsupportedOperationException(...)`,
   напиши конструктор, геттеры, сеттеры с валидацией, `toString`/`equals`/`hashCode`.
6. Запусти тесты задачи снова — добейся зелёного.
7. Закоммить и переходи к следующей задаче.

## Правила оформления классов

- Класс публичный, пакет `com.course.tasks`.
- Поля `private`. `final`, если поле не помечено как «изменяемое».
- Конструктор заполняет все поля (с валидацией, где указано).
- Геттеры на все поля. Сеттеры — только на «изменяемые» поля, с валидацией.
- Невалидные данные → `IllegalArgumentException` с описательным сообщением.
- `toString`, `equals`, `hashCode` переопределены всегда.
